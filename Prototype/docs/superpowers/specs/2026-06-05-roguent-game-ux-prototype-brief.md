# Roguent 游戏化 UX 原型设计 brief

> 这份文档是给 **Claude Design** 的输入。目标:产出一套**高完成度、美观、像素游戏风**的桌面应用 UI 原型,把 Claude Code / Codex 的真实 agent 活动渲染成"一屋子小人在地牢里干活"。请把它当作**风格 + 信息架构 + 逐面板规格**的完整说明书来设计;3 张《元气骑士》截图作为美术基调参考(随附)。

---

## 1. 一句话定位

**Roguent = 本地 Claude Code + Codex 双 runtime 的中央调度平台**。用户在这里同时看着 / 指挥多个 AI 编码会话;每个会话 = 一个《元气骑士》风格的像素小人(NPC)在地牢房间里干活。它既是"实时可视化大屏",也是"可操作的指挥台"——所有面板(任务、技能、商店、设置、排行榜)都套上元气骑士的游戏化皮肤。

**平台**:macOS 桌面应用(Tauri / WKWebView)+ 浏览器。**不是手游**——截图里的虚拟摇杆/瞄准盘要去掉,改桌面交互(见 §8)。

---

## 2. 原型范围 & 交付物

请设计**一套统一视觉语言下的桌面应用原型**,包含:

- **1 个主画面(游戏世界 + HUD)**:像素地牢房间俯视图 + 边缘叠加的全套 HUD(限额条、货币、按钮坞、底部 hotbar、NPC 头顶状态)。
- **约 10 个弹窗/全屏面板**(由 HUD 按钮唤起,见 §6.7–6.14):NPC 个人卡片、任务面板、设置面板、系统菜单、排行榜、技能面板、商店(插件市场 + 道具店)、压缩阈值设置等。
- **2 个过场/状态画面**:加载/过场(蓝色漩涡)、空态/错误态。

视觉上要像**一款打磨过的独立像素游戏**,不是"管理后台套了像素皮"。优先做到:统一的木框面板 chrome、像素字体、霓虹强调色、克制的动效。所有面板共享同一套设计 token(§9)。

---

## 3. 美术方向(从 3 张截图提炼)

### 3.1 总基调
- **视角**:top-down 俯视像素地牢 / 城镇,像《元气骑士》(Soul Knight)。
- **画风**:16px 网格像素美术,角色是 Q 版小人(大头、2~3 帧待机/行走)。
- **氛围**:地牢偏深青蓝 + 暗角(vignette),城镇偏暖棕木质 + 绿植;整体暗背景让霓虹色发光元素跳出来。

### 3.2 配色(从截图取色,见 §9 token)
- **底色**:深青灰地砖(`#1f2e33`~`#2a3d44`)、暖棕木板(`#5a3f2a`~`#3a2718`)、近黑暗角(`#0b0a12`)。
- **强调霓虹**:青 `#36c5e0`、粉红/红 `#ff4d6d`、金 `#f2c84b`、绿 `#5fd35f`、紫 `#a06cd5`。
- **血条/蓝条**:红(HP/5h 限额)、灰蓝(护盾)、宝石蓝(MP/周限额)——见 §6.1。

### 3.3 UI chrome(关键,务必还原截图质感)
- **面板**:暖棕木质底 + **多层像素斜角描边**(外深棕 → 内浅棕高光),角落有小铆钉/装饰点(参考截图 ① 左上血条框、活动卡框,以及截图 ② 商店摊位)。
- **标题栏**:面板顶部一条深色标题带 + 像素图标 + ✕ 关闭按钮。
- **按钮**:方形像素按钮,带"凸起"高光与按下"凹陷"态;选中/激活态点亮描边(青或金辉光)。
- **字体**:像素点阵字体(项目已用 `Press Start 2P`);中文用等宽像素中文字体兜底(如"Fusion Pixel"类),保证中英混排不串位。
- **图标**:**全部用自定义像素图标,禁止使用系统 emoji**(emoji 太粗糙、风格不统一)。图标/按钮系统见 §3.5,角色精灵见 §3.6。

### 3.4 从 3 张截图各自要借鉴的元素
- **截图 ①(实验室房间)**:左上**血条/护盾/蓝条三条堆叠** + 数值(7/7、6/6、180/180);其下**"活动"卡**(轮播一排角色头像,带金色高光边);右上**货币计数**(🐟10 / 💎14099 / 🪙0,图标+数字);右上 ⏸ 暂停 + 竖排方形图标按钮;地上**发光物品箱/基座**、电脑/打印机道具、黑猫宠物。
- **截图 ②(城镇/商店)**:一群**站桩 NPC**(法师、精灵、骑士、忍者等不同皮肤)围坐/站立;**自动贩卖机、扭蛋机、商品摊位、发光书本基座、带翅膀的"X"祭坛传送门**;**宠物**(黑猫、狗);右上有**对话/信件图标**(💬/✉)。→ 城镇/商店氛围 + NPC 多样性 + 传送门。
- **截图 ③(蓝色漩涡过场)**:全屏深色,中央**发光蓝色漩涡(粒子)**,顶部大字提示("1个关卡后获得新天赋!"),底部小字副提示,右下 **💾 存档图标**。→ 加载/过场/解锁画面的范式。

### 3.5 自定义图标 / 按钮系统(取代 emoji)
**原则:全程使用一套自绘像素图标,绝不出现系统 emoji。** 本 brief 行文里出现的 ❤ / 💎 / 📜 等 emoji 只是**语义占位**(指代该图标表达什么),最终一律换成下面这套自绘像素图标。

**图标绘制语言**
- 画在 **16×16 / 24×24 像素网格**上,**硬边、无抗锯齿、无渐变**(需要过渡用像素抖色 dithering)。
- 每个图标:**1–2px 深色描边**(`#2c1c10`)+ 2–3 个平涂色块 + 1 处左上高光 + 1 处右下阴影(**统一光源:左上**)。
- 走 **RPG 道具隐喻**,不用现代扁平/线性 UI 图标:用"剑 / 法术书 / 背包箱 / 对话卷轴 / 宝石 / 齿轮符文 / 奖杯 / 摊位旗"这类地牢词汇。
- 配色从 §9 强调色取;同一面板内图标色相克制(2~3 个主色),靠**造型剪影**而非颜色区分。

**图标钮四态**
- **默认**:木质凸起方钮(§3.3 chrome),图标居中带 1px 投影。
- **hover**:整体上浮 1px + 描边提亮。
- **激活 / 选中**:钮面内凹 + 一圈强调色辉光描边(色相随图标语义,如技能青、商店金)。
- **禁用**:去饱和 + 叠一把小锁;**未读 / 有事**:右上角红点角标。
- 两种尺寸:hotbar / 按钮坞渲染 32–40px(图标 24px),行内 / 数值旁 16px。

**图标清单(自绘,括号内是造型,非 emoji)**
- HP/5h(红心或红药瓶)、周/MP(蓝色法力晶体)、token(金币堆)、宝石货币(切面宝石)、完成数(月桂 / 对勾旗)。
- 技能(法术书)、背包(皮袋 / 宝箱)、聊天(对话卷轴)、模型(多面水晶 / 脑芯)、导入(文件夹+箭头 / 卷轴传送)、任务(任务卷轴 / 羊皮夹)、商店(摊位遮阳棚旗,**不用购物车**)、排行榜(奖杯+领奖台)。
- 设置(齿轮符文)、菜单(三道符文横条)、账号(骑士头像框)、存档(宝箱 / 像素软盘符文,呼应截图③但重绘)。
- 工具图标:Read=翻开的书、Write/Edit=羽毛笔 / 凿子、Bash=试管药剂、Search=放大镜、Task=魔杖+星、MCP=插头 / 接口。
- 状态:askuser=**发光"?"符文嵌对话泡(最醒目)**、待办=小卷轴、idle=漂浮 zzz、done=绿色对勾、error=红色火花 / 骷髅。
- runtime 徽:**Claude=蓝色符文印记**、**Codex=绿色括号 / 终端符文**(两枚一眼可辨的徽章)。

### 3.6 角色 / 精灵设计(取代默认头像,沿用 0x72 风骨)
现有运行时用的是 **0x72 DungeonTilesetII** 像素角色(knight / wizard / elf / lizard / dwarf…,见 `src/shared/mapping.ts` 的 `HERO_POOL` / `ORCHESTRATOR_HERO`)。**自绘角色要与这套 0x72 风骨同语言**(同比例、同描边、同调色盘),好让原型能平滑落到真实渲染,而不是另起炉灶。

**比例与帧**
- Q 版,约 **2 头身**,大轮廓、强剪影,小尺寸也一眼可辨身份(元气骑士式)。基准 **16×16 / 16×24**。
- 帧:待机(2 帧呼吸 / 眨眼)、行走(4 帧)、工作(1–2 帧"凑近工具 / 伏案"姿势)、受击 / 出错(红闪 1 帧)。动画预算克制,留复古手感。

**角色梯队**
- **主控 ★(orchestrator)**:金色骑士,略大一号 + 披风 / 冠饰,常驻房间 C 位。
- **subagent / teammate**:从英雄池(法师 / 精灵 / 蜥蜴人 / 矮人 / 女骑士 / 兽人…)按 role 哈希取**稳定皮肤**,一角色一配色,让"谁是哪个角色"可记忆。
- **runtime 区分**:即便同一原型角色,也用**肩甲 / 披风色(Claude 蓝 / Codex 绿)+ 头顶漂浮 runtime 符文**(§3.5 徽)区分,不靠换造型。

**姿态即状态**(配合 §6.6 头顶图标槽)
- idle 放松站立;working 朝工具 / 书桌倾身;thinking 头顶"…"光点;askuser **转身正对镜头 + 头顶"?"符文**;done 举手 / 对勾;error 红闪 / 冒烟。

**宠物**:1×1 小生物(黑猫 / 狗 / 史莱姆),2 帧待机,跟随主控;纯陪伴,无功能影响。

**统一描边 / 上色**:与图标系统同规(深色描边、限色、左上单光源),整体像同一支美术之手画出来的。

│      🐈‍⬛宠物            [📋]                  🛡小人                          │
│                  ← 像素地牢房间(俯视,网格地砖,门口,道具) →                   │

│  [小地图]                                                       [选中NPC高亮] │
│                                                                            │

- **房间是舞台**:中央是当前选中会话的地牢房间;主控是金骑士★,subagent / teammate 是各色小人,会走动、转向、头顶冒工具气泡。
- **HUD 叠在四周**,半透明木框面板,不挡住房间中心。
- **双层缩放**(已实现):大厅"总览世界"(多项目 = 多房间,会话 = 站桩 NPC)↔ 进入某会话的"内景房间"。本原型主要画**内景 + 大厅各一版**。

## 6. 逐面板详细规格

> 每个面板都套 §3.3 的木框 chrome。下面给"内容 + 状态 + 关键交互",视觉细节请按截图基调发挥。
> **注意:下文出现的 emoji(❤/💎/📜/🪄… 等)只是语义占位,标明该处图标"代表什么";最终一律改用 §3.5 的自绘像素图标,不渲染 emoji。**

### 6.1 左上 · 账户限额双条(已实现雏形,需美化)
- 两条堆叠,**条长 = 剩余额度**(`100 - 已用%`),随消耗见底:
  - **5h 限额** = 红血条(❤ 图标)。
  - **周限额** = 蓝魔法条(💎 图标)。
- 旁注小字:**plan 名**(Pro / Max / Team)+ **重置倒计时**(如 `2h13m`)。
- 剩余低于 15% → 变警示红 + 轻微闪烁。
- 同步中/取数失败 → 灰态 + "同步中 / ⚠"小字(不报错弹窗)。
- 参考截图 ① 左上三条的框体质感(我们用两条;若要凑足三条的视觉密度,可把"当前选中会话上下文%"做成第三条灰蓝护盾条作呼应,但语义上账户级只有 5h/周两条)。

### 6.2 左上 · "在岗"轮播卡(对应截图 ① "活动")
- 限额条下方一张小卡:**横排活跃会话 NPC 头像**(最多 5~6 个),金色高光边轮播;角标显示"X 在岗"。
- 点头像 = 把渲染源切到该会话(相机进该房间)。
- 有 ❓/出错的 NPC 在此卡上也打对应角标,方便"哪个会话在等我"。

### 6.3 顶部中央 · 会话横幅 + 右上货币计数(已实现雏形)
- **中央横幅**:`⚔ 会话标题 · 模型简称 · NP(几个 agent)· 🪙token`。
- **右上货币计数**(对应截图 ① 右上 🐟/💎/🪙),映射真实指标,带图标:
  - 🪙 **token**:当前/累计 token 用量。
  - 💎 **宝石**:道具店货币(完成会话/任务赚取,见 §6.14)。
  - ✅ **完成数**:已完成会话/任务计数(成就感指标)。
- **runtime 指示**:右上一个 `☁ Claude | ⌗ Codex` 切换/筛选片(见 §6.16)。

### 6.4 右上 · 按钮坞(设置 / 菜单 / 账号)
竖排方形像素按钮(对应截图 ① 右上按钮组):
- **⚙ 设置** → §6.10 配置面板。
- **☰ 菜单** → §6.11 系统/暂停菜单。
- **👤 账号** → 订阅/login 态弹窗(plan、用量、登出)。
- **⏸ 暂停**(可选,过场/聚焦用)。

### 6.5 底部中央 · hotbar(已实现 6 格,新增 2 格)
横排像素图标按钮,激活态点亮描边。分两组用细分隔:
- **会话操作组**:📜技能 · 🎒背包 · 💬聊天 · 💎模型 · 📂导入会话。
- **系统组**:📋任务 · 🛒商店 · 🏆排行榜。
- 每格 hover 出像素 tooltip;有未读(如新 loot、新 askuser)时角标红点。

### 6.6 NPC 头顶状态(主画面内,最关键的"实时感"来源)
每个 NPC 头顶**从上到下**:
1. **昵牌**:角色名/role(小像素标签底)。
2. **上下文充能条**:细条,**填充=已用 utilization%**;`<20%`绿 / `20–80%`琥珀 / `>80%`红;**20% 处一道竖刻度线**(= 默认压缩阈值,见 §6.8)。
3. **状态图标槽**(任一时刻只显其一,优先级从高到低):
   - **❓**(高亮闪烁)= 调了 askuser、在等用户回应 → **最高优先,必须醒目**。
   - **📋** = 有 pending task/待办。
   - **工具图标**(📖/⌨️/🧪/🔍/🪄/🔌)= 正在调用该工具。
   - **待命表情 / 💤** = idle。
   - **✓** 绿 = done;**红灯/💢** = error。
- **runtime 徽标**:NPC 脚下/肩上一个小徽(Claude 蓝 / Codex 绿)。

### 6.7 NPC 个人卡片(点 NPC 弹出)

### 6.8 压缩阈值 + 自动编排(功能 3 + 11,核心差异化)
**动机(请在 UI 里用一句 tooltip 解释)**:订阅模式下 Opus 默认 1M 上下文、改不回 200k,不设阈值会烧爆额度;所以让用户为每个模型设"达到 X% 就自动压缩续跑"。

**两级设置,局部 > 全局**:
- **全局默认**(在 §6.10 设置面板内,一个"上下文压缩"分组):对 **3 个模型**分别设阈值——
  - `claude-opus-4-8`:默认 **20%**。
  - `claude-sonnet-4-6`:默认 **Auto**。
  - `claude-haiku-4-5`:默认 **Auto**。
  - ("Auto" = 不主动干预,走 SDK 原生到上限自动压缩;只有设了百分比的模型走主动编排。)
- **局部覆盖**(在 §6.7 NPC 卡片内):该 NPC 单独设阈值,**优先级高于全局**;UI 上显示"继承全局 20% / 已覆盖为 N%"。

**控件外观**:百分比滑块 + 数字输入 + "Auto"切换;滑块上标 20% 默认刻度;旁边 ❓ tooltip 讲清流程。

**自动编排流程(用一个像素"流程卡"图示展示这 4 步循环)**:
当某 NPC 上下文 `utilization ≥ 阈值`(如 Opus 到 20%,即约 200k):
1. **终止**当前任务/本轮。
2. 发送并触发 **`/compact`**,监控等待压缩成功(头顶条回落)。
3. 压缩成功后**发送"继续"prompt**,接着干。
4. **循环**:只要再次越过阈值就重复 1–3。
- UI 反馈:编排进行时 NPC 头顶/卡片显示"🔄 压缩中…"状态;Auto 模型不触发此流程。

### 6.9 任务面板(功能 5 + 7 + 9 + 10 · CC 真任务)

- **数据来源**:聚合各会话的 **TodoWrite 待办** + **subagent task** + **agent team 共享任务清单**(team config `~/.claude/teams/`、task list `~/.claude/tasks/`),**实时同步状态**。
- **任务状态三态**(请用不同色像素徽表示):`pending`(待领,灰)/ `in progress`(进行中,青,带 spinner)/ `completed`(完成,绿✓)。
- **任务依赖**:被依赖未完成的 pending 任务显示"🔒 阻塞中"(依赖完成后自动解锁)。
- **认领(claim)**:显示任务归属哪个 NPC(头像角标);未认领任务可见"待认领"。
- **mailbox / 协作**:teammate 之间互发消息——用一条"信箱"小区展示最近 inter-agent 消息(谁 → 谁:一句话)。

- **布局建议**:左列 = 任务清单(按状态/会话分组,可折叠);右列 = 选中任务详情(描述、归属、依赖、状态时间线)。
